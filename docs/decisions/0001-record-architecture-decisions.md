# 0001. Record Architecture Decisions

## Status

Accepted

## Context

OSIPY Dashboard contains frontend architecture, deployment, workflow, and future data-handling decisions that future contributors need to understand.

## Decision

Significant technical, workflow, deployment, and data-handling decisions will be documented as Architecture Decision Records in `docs/decisions/`.

## Consequences

- Pull requests that introduce major architecture, dependency, deployment, persistence, or OSIPY runner integration changes should add or update an ADR.
- Meeting or chat decisions should be summarized in durable repository documentation.
- Future maintainers can reconstruct why important choices were made.
